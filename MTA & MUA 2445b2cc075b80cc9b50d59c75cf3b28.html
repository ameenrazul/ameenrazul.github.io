MTA & MUA

📧 MTA vs MUA (Email Components)

TermFull FormRoleYou interact with it?
MUAMail User AgentThe email app you use to send/read emails✅ Yes
MTAMail Transfer AgentThe email server that routes/delivers the email❌ No (usually behind the scenes)

📱 1. MUA – Mail User Agent

What it is:

A Mail User Agent is the program you use to send and read emails.

✅ Examples:

  • Microsoft Outlook
  • Thunderbird
  • Apple Mail
  • Gmail app / webmail
  • K-9 Mail (Android)

🔹 Functions:

  • Compose and send emails
  • Read and organize inbox
  • Connect to server using SMTP, IMAP, or POP3

🔧 Protocols used:

  • SMTP – for sending
  • IMAP/POP3 – for receiving

📤 2. MTA – Mail Transfer Agent

What it is:

A Mail Transfer Agent is the server-side software responsible for transferring emails from sender to recipient over the internet.

✅ Examples:

  • Postfix
  • Sendmail
  • Exim
  • Microsoft Exchange Transport Service

🔹 Functions:

  • Accepts emails from MUAs or other MTAs
  • Looks up DNS MX records to find destination mail server
  • Forwards the email to next MTA or final destination

🔧 Protocol used:

  • SMTP – Simple Mail Transfer Protocol

🔄 How MUA and MTA Work Together:

When Alice sends an email to Bob:

  1. Alice writes email in her MUA (e.g., Outlook).
  1. The MUA sends it via SMTP to her outgoing MTA (e.g., smtp.example.com).
  1. The MTA forwards it across the internet, possibly via other MTAs.
  1. It finally reaches Bob's MTA, which stores it.
  1. Bob's MUA (e.g., Gmail) retrieves it using IMAP or POP3.
  1. Bob reads the email.

               (Send)         (Route)         (Receive)         (Read)
