MISP

📌 What is MISP?

MISP (Malware Information Sharing Platform) is an open-source threat intelligence platform designed to:

  • Collect, store, and share cyber threat intelligence (CTI)
  • Distribute Indicators of Compromise (IOCs) such as:
    • Malware hashes
    • IP addresses
    • URLs/domains
    • Tactics, Techniques, and Procedures (TTPs)

It helps analysts, researchers, SOC teams, law enforcement, and threat intelligence communities collaborate more effectively.

🔄 MISP Sharing Model

MISP uses a distributed model:

  • Supports private, semi-private, or public sharing
  • Can sync with other MISP instances
  • Allows automated sharing across trusted communities

⚙️ Core Functionalities

1. IOC Database

  • Central storage for technical (e.g., IPs, hashes) and non-technical data (e.g., threat actor names)

2. Automatic Correlation

  • Detects relationships between events, attributes, and campaigns

3. Data Sharing

  • Securely share intelligence across MISP instances and user groups

4. Import & Export

  • Import/export in various formats:
    • STIX, OpenIOC, Suricata, Snort
    • Integrates with NIDS, SIEMs, and HIDS

5. Event Graph

  • Visual map of how events, objects, and attributes are linked

6. API Integration

  • RESTful API for automation and integration with other platforms

📝 Event Management in MISP

Events are containers for linked intelligence on malware, attacks, and investigations.

🛠️ 1. Event Creation

Create an event by providing:

  • Title/Description
  • Date
  • Threat Level (Low / Medium / High / Undefined)
  • Distribution Level (Who can see this?)

📤 Distribution Levels:

OptionVisibility Scope
Your Organisation OnlyInternal users only
This Community OnlyLocal MISP server & connected orgs
Connected CommunitiesLocal + 1-2 hop away synced servers
All CommunitiesPublic propagation to all MISP nodes
Sharing GroupCustom list of trusted orgs

✍️ 2. Adding Attributes & Attachments

➕ Add Attribute:

  • Manual input (single IOC like IP, hash, URL)
  • Batch import (e.g., multiple IPs via line break)
  • Import from external formats (OpenIOC, ThreatConnect)

🛡️ Special Options:

SettingUse
For Intrusion Detection SystemMarks attribute as usable by IDS (e.g., Suricata/Snort)
Batch ImportPaste multiple IOCs at once
Malware CheckboxFor attachments; ZIPs + passwords them to prevent misuse

📎 Add Attachment:

  • Upload malware samples, reports, binaries, dropped files
  • Mark file as malware (adds safe packaging)

Example: Upload CobaltStrike.exe and tag it as malware.

📣 3. Publish Event

  • Events are not shared until published.
  • Publishing adds them to:
    • Your MISP instance's event pool
    • Connected communities or groups (based on distribution setting)
  • Typically reviewed by org admin before publishing
