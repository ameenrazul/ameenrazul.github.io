Redline

Redline, a tool by FireEye, is designed to help security analysts and incident responders quickly assess potentially compromised endpoints. While powerful tools like Volatility offer deep memory analysis, they can be time-consuming. Redline provides a faster, high-level overview of Windows, Linux, or macOS systems.

Key Features of Redline:

  • Collects registry data (Windows only)
  • Gathers running processes
  • Captures memory images (pre-Windows 10)
  • Retrieves browser history
  • Detects suspicious strings
  • Offers a user-friendly graphical interface for analysis

Let's move to the Data Collection stage.¬†There are three ways or options to collect data using Redline:

Redline offers three types of collectors to gather data from endpoints:

  1. Standard Collector
    • Purpose: Gathers minimal but essential data for quick analysis
    • Use Case: Preferred for triage and rapid assessment
    • Duration: Fastest option ‚Äî only takes a few minutes
  1. Comprehensive Collector
    • Purpose: Gathers a full range of data from the system for in-depth analysis
    • Use Case: Chosen when a thorough investigation is needed
    • Duration: Time-consuming ‚Äî can take an hour or more
  1. IOC Search Collector (Windows only)
    • Purpose: Collects data matching known Indicators of Compromise (IOCs)
    • Use Case: Used when you have predefined IOCs via the IOC Editor, often sourced from threat intel or prior analysis
    • Requirement: IOC Editor must be used to define and import IOCs before scanning

Each method serves a different need, from rapid response to full forensic investigation.

The system will provide you with machine information:

  • Machine and operating system (OS) information
  • Analyze system restore points (Windows versions before 10 only)
  • Enumerate the registry hives (Windows only)
  • Obtain user accounts (Windows and¬†OS¬†X only)
  • Obtain groups (OS¬†X only)
  • Obtain the prefetch cache (Windows only)

Network:

Network Options supports Windows,¬†OS¬†X, and¬†Linux¬†platforms. You can configure the script to collect network information and browser history, which is essential when investigating the browser activities, including malicious file downloads and inbound/outbound connections.

Other:

Summary of Redline Analysis View:

When data is imported into Redline, the left panel shows various Analysis Data categories, helping analysts investigate a potentially compromised system.

üîç Key Analysis Sections:

  • System Information

    Displays machine details including BIOS (Windows only), OS, and user accounts.

  • Processes

    Includes attributes like process name, PID, path, arguments, parent process, and user. Expanding a process shows:

    • Handles: Links between processes and system resources (e.g., files, registry).
    • Memory Sections: Unsigned memory sections can indicate malicious DLLs.
    • Strings: Captured in-memory strings for each process.
    • Ports: Reveals network activity‚Äîlook for unusual outbound connections, especially from system processes (e.g., explorer.exe with network activity is suspicious).

üõ† Other Important Sections:

  • Registry: Can reveal persistence mechanisms and configuration changes.
  • Windows Services: Threat actors may install malicious services.
  • Tasks: Scheduled tasks are often used for persistence.
  • Event Logs: Look for PowerShell activity, logon/logoff events, user creation, etc.
  • File Download History: Shows downloaded payloads or tools.
  • File System
  • ARP and Route Entries
  • Browser URL History

üìÖ Timeline View:

  • Tracks file and process activity (create, modify, access, change)
  • Helps reconstruct the attacker‚Äôs steps and the timing of compromise/escalation activities.

This structured view helps quickly spot indicators of compromise and understand attack flow.

If you know when the host compromise or suspicious activity occurred, you can use¬†¬†TimeWrinkles‚Ñ¢¬†¬†to filter out the timeline to only the events that took place around that time.

TimeCrunches‚Ñ¢¬†¬†helps to reduce the excessive amount of data that is not relevant in the table view. A TimeCrunch will hide the same types of events that occurred within the same minute you specified.
