What is "Threat Modelling”

  1. Identifying what systems and applications need to be secured and
    what function they serve in the environment. For example, is the system
    critical to normal operations, and is a system holding sensitive
    information like payment info or addresses?
  1. Assessing what vulnerabilities and weaknesses these systems and
    applications may have and how they could be potentially exploited
  1. Creating a plan of action to secure these systems and applications from the vulnerabilities highlighted
  1. Putting in policies to prevent these vulnerabilities from occurring
    again where possible (for example, implementing a software development
    life cycle (SDLC) for an application or training employees on phishing awareness).

STRIDE, DREAD and CVSS (to name a few) are all frameworks specifically used in threat modelling

Introducing the Unified Kill Chain

Phase: In (Initial Foothold)

🔍 Reconnaissance (MITRE TA0043)

This is the information-gathering phase.

An attacker collects details about their target using:

  • 🔸 Passive methods (e.g. looking up public info).
  • 🔸 Active methods (e.g. scanning systems directly).

The info gathered here helps in all later attack stages.

📌 They might find:

  • What systems/services are running (useful for later attacks).
  • Contact or employee lists (useful for phishing or impersonation).
  • Credentials (to access systems later).
  • Network layout (to plan internal movement or pivoting).

💣 Weaponization (MITRE TA0001)

In this phase, the attacker prepares tools and infrastructure to launch their attack.

📌 Examples:

  • Setting up a Command & Control (C2) server.
  • Preparing systems to catch reverse shells or deliver malware.

🕵️ Social Engineering (MITRE TA0001)

The attacker manipulates people to help them gain access or information.

📌 Examples:

  • Tricking a user into opening a malicious file.
  • Making a fake login page to steal passwords.
  • Calling or visiting in person, pretending to be someone (e.g. IT or a utility worker).

💥 Exploitation (MITRE TA0002)

This is where the attacker uses vulnerabilities to run code or take control of a system.

📌 Examples:

  • Uploading and running a reverse shell through a website.
  • Modifying scripts already on the system to run attacker’s code.
  • Exploiting a website vulnerability to gain system access.

🔁 Persistence (MITRE TA0003)

The attacker wants to stay inside the system, even after a reboot or being kicked out.

📌 Examples:

  • Creating a hidden service to allow re-entry.
  • Connecting the target system to their C2 server for ongoing access.
  • Installing a backdoor that triggers on specific events (like admin login).

🛡️ Defence Evasion (MITRE TA0005)

The attacker hides their presence to avoid detection.

📌 They try to bypass:

  • Web application firewalls (WAFs)
  • Network firewalls
  • Antivirus software
  • Intrusion detection systems (IDS)

🎯 This helps security teams improve their defences after learning what was bypassed.

🎮 Command & Control (MITRE TA0011)

Here, the attacker communicates with the compromised system.

📌 What they can do:

  • Run commands.
  • Steal data, passwords, and other sensitive info.
  • Use the compromised system to move further into the network (pivoting).

🔄 Pivoting (MITRE TA0008)

The attacker moves from the initially hacked system to other internal systems that aren't directly accessible (e.g. not exposed to the internet).

📌 Example:

  • They hack a public-facing web server, then use it to attack internal servers that are on the same network.

🔄 Pivoting (MITRE TA0008)

After getting access to a system, the attacker uses it as a base of operations inside the victim's network.

📌 That system acts as:

  • A tunnel between the attacker and the network.
  • A staging area to launch attacks.
  • A distribution point to send malware or install backdoors to other systems.

🧭 Discovery (MITRE TA0007)

The attacker gathers detailed information about the compromised system and its network.

📌 They look for:

  • Active user accounts and their permissions.
  • Installed applications and software.
  • Web browser activity and files.
  • System configurations.
  • Network shares, files, folders, and connected devices.

🎯 This helps the attacker plan their next moves.

🔐 Privilege Escalation (MITRE TA0004)

The attacker tries to gain higher-level permissions on the system by exploiting what they discovered earlier.

📌 They aim to become:

  • SYSTEM (Windows) or ROOT (Linux)
  • A Local Administrator
  • A user with Admin-like powers
  • A user with special roles (e.g. backup admin, domain-specific roles)

🔧 They do this by abusing:

  • Misconfigurations
  • Vulnerable accounts
  • Unpatched software

💻 Execution (MITRE TA0002)

The attacker now runs malicious code on the compromised system.

📌 This could include:

  • Remote access trojans (RATs)
  • Command & Control (C2) scripts
  • Malicious links or documents
  • Scheduled tasks to ensure continued access

🎯 This supports persistence and further spread.

🧪 Credential Access (MITRE TA0006)

The attacker tries to steal usernames and passwords to move around undetected.

📌 Methods used:

  • Keylogging
  • Credential dumping (extracting passwords stored in memory or system files)
  • Grabbing browser-stored credentials

🎯 Using real credentials makes them harder to spot.

🧬 Lateral Movement (MITRE TA0008)

Now armed with credentials and high privileges, the attacker moves to other systems inside the network.

📌 They:

  • Jump from one system to another.
  • Look for higher-value targets (like servers or databases).
  • Use stealthy methods to avoid detection (e.g., using built-in tools like PsExec, RDP, or remote WMI).

🎯 The goal is to get closer to their main objective, like stealing sensitive data or taking control of critical infrastructure.

Phase: Out (Action on Objectives)

📥 Collection (MITRE TA0009)

After gaining access and control, the attacker now focuses on gathering valuable data.

📌 They collect from sources like:

  • Drives (local or network)
  • Web browsers
  • Audio and video files
  • Email systems

🎯 This phase breaks confidentiality and prepares the attacker for Exfiltration.

📤 Exfiltration (MITRE TA0010)

Now the attacker tries to steal the collected data and send it out of the network without being detected.

📌 Techniques used:

  • Encrypting and compressing the data to hide its contents.
  • Sending data through the C2 tunnel established earlier.

🎯 This helps bypass firewalls, intrusion detection, and logging systems.

💣 Impact (MITRE TA0040)

At this stage, the attacker may want to cause damage or disruption by affecting data integrity (correctness) and availability.

📌 Methods include:

  • Deleting user accounts or changing permissions.
  • Wiping disks or critical files.
  • Encrypting files with ransomware.
  • Website defacement (changing content to embarrass or threaten).
  • Denial of Service (DoS) to bring down services or networks.

🎯 Goal: Disrupt the business or organization severely.

🎯 Objectives

With full control and access, the attacker now executes their final mission—the real reason behind the attack.

📌 Examples:

  • 💰 Financial: Deploy ransomware, demand payment for data recovery.
  • 🕵️‍♂️ Reputation damage: Leak private/confidential information to the public.
  • 📉 Operational disruption: Shut down critical services or systems.

🎯 This is where the attack fulfills its main strategic goal.
