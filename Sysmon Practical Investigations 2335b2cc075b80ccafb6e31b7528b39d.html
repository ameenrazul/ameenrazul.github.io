  1. What is the device name when being called by RawAccessRead in Investigation 1?

https://www.ultimatewindowssecurity.com/securitylog/encyclopedia/event.aspx?eventid=90009

In Sysmon, the RawAccessRead event (Event ID 9) indicates that a process has performed a low-level read operation directly from a physical disk. This is not the usual way applications read files—they typically go through the file system via the Windows API (e.g., using ReadFile() or CreateFile() on logical volumes). A RawAccessRead bypasses that and accesses the disk at the raw level, which is very uncommon and often considered suspicious.
