Windows Event Logs

‚úÖ What Are Event Logs?

  • Event logs are system-generated records of noteworthy activities like:
    • Authentication attempts (successful/failed)
    • System startup/shutdown
    • Application crashes
    • Service status changes

üìå Use Cases:

  • System admins / engineers use them for:
    • Troubleshooting crashes and performance issues.
    • Auditing system behavior.
  • Blue teamers / defenders use them for:
    • Investigating suspicious activity.
    • Detecting lateral movement, privilege escalation, persistence, etc.

üîç Why Do Logs Matter to Defenders?

  • Attackers leave traces, even if subtle:
    • Failed logins (Event ID 4625)
    • Privilege use (Event ID 4672)
    • Scheduled tasks (Event ID 4698)
  • Correlating logs across hosts (via SIEM) helps detect larger patterns, like:
    • Brute force attempts across different servers.
    • Coordinated malware execution.

Get-WinEvent

Get-WinEvent PowerShell Cmdlet ‚Äî Key Points

  • Purpose: Retrieves events from event logs and event tracing files on local or remote Windows computers.
  • Replaces: Get-EventLog cmdlet (more modern, versatile).
  • Capabilities:
    • Query multiple event logs at once.
    • Filter events using XPath, structured XML, or hash tables (FilterHashtable parameter).

How to Get Help

  • Use Get-Help Get-WinEvent or visit Microsoft docs online for detailed usage info.

Example Usage

  1. List all event logs on a computer:
    Get-WinEvent -ListLog *

    Output includes log name, size, type (circular, etc.), and record count.

  1. List event log providers and associated logs:
    Get-WinEvent -ListProvider *

    Shows providers (e.g., .NET Runtime) and the logs they write to.

  1. Filter event logs by provider name:

    Using pipeline filtering (less efficient):

    Get-WinEvent -LogName Application | Where-Object { $_.ProviderName -Match 'WLMS' }

    Using recommended FilterHashtable parameter (more efficient):

    Get-WinEvent -FilterHashtable @{ LogName='Application'; ProviderName='WLMS' }

Hash Table Syntax for Filtering

  • Format:
    @{ Key1 = Value1; Key2 = Value2; ... }
  • Guidelines:
    • Start with @
    • Enclose in {} braces
    • Separate key-value pairs with =
    • Use semicolons or new lines to separate pairs
  • Example:
    @{ LogName='Application'; ProviderName='WLMS' }

Useful Tips

  • For large logs, prefer FilterHashtable over Where-Object for better performance.
  • Event Viewer GUI helps identify log names and providers to build filters.
  • Example advanced filter:
    Get-WinEvent -FilterHashtable @{LogName='Microsoft-Windows-PowerShell/Operational'; ID=4104} |
  Select-Object Message |
  Select-String 'SecureString'

That‚Äôs the essence‚ÄîGet-WinEvent is powerful and flexible for working with Windows event logs, especially when using the FilterHashtable for efficient filtering.

XPATH

üõ†Ô∏è XPath Filtering with Get-WinEvent

üîπ What is XPath?

  • XPath (XML Path Language) is a W3C standard used to navigate and filter XML documents.
  • Windows Event Logs support a subset of XPath 1.0.
  • Used in Event Viewer, Get-WinEvent, and wevtutil.exe to create powerful event filters.

üîç Basic XPath Syntax

Get-WinEvent -LogName <LogName> -FilterXPath '<XPath query>'
  • Starts with or Event
  • Use / to navigate XML structure like System, EventID, Provider, etc.

üß™ Example 1: Events with Severity ‚â§ 3 in Last 24 Hours

*[System[(Level <= 3) and TimeCreated[timediff(@SystemTime) <= 86400000]]]

üß™ Example 2: EventID = 100 from Application Log

Get-WinEvent -LogName Application -FilterXPath '*/System/EventID=100'

üß™ Example 3: Filter by Provider Name (WLMS)

Get-WinEvent -LogName Application -FilterXPath '*/System/Provider[@Name="WLMS"]'

üß™ Example 4: Combine Multiple Conditions (Provider + EventID)

Get-WinEvent -LogName Application -FilterXPath '*/System[EventID=101 and Provider[@Name="WLMS"]]'

üß™ Example 5: Filter EventData Fields

Query for specific data inside <EventData> section (e.g., TargetUserName = System):

Get-WinEvent -LogName Security -FilterXPath '*/EventData/Data[@Name="TargetUserName"]="System"' -MaxEvents 1

üß™ Using XPath with wevtutil.exe

wevtutil.exe qe Application /q:"*/System[EventID=100]" /f:text /c:1
  • /q: is the XPath filter
  • /f:text = plain text output
  • /c:1 = return only 1 event

üß† Tips

  • Use Event Viewer ‚Üí Details ‚Üí XML View to inspect XML structure and build XPath.
  • Use /System/... for common filters like EventID, Provider, TimeCreated.
  • Use /EventData/Data[@Name="FieldName"] for filtering custom values like usernames or IPs.
  • MaxEvents <n> limits number of returned events.

üìò Further Reading

  • Microsoft official docs: XPath Syntax Reference

This summary gives you everything you need to construct and understand XPath queries for Get-WinEvent or wevtutil.
