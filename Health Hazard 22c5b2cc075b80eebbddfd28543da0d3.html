Phase 1: Initial Compromise & Script Execution Trigger

  • Title: Malicious Package Installation and Post-Install Script Execution
  • Adversary Step Description: The user PAW-TOM\itadmin-tom executes the command npm install healthchk-lib@1.0.1 within the C:\Development\ directory. This command is run by node.exe, which was itself launched by a PowerShell process (PID 1452). During the installation process, node.exe creates the malicious postinstall.ps1 script file within the package's directory (C:\Development\node_modules\healthchk-lib\scripts\). Immediately following this, the node.exe process executes the postinstall script by launching a cmd.exe process.
  • Timestamp (Observed): 21/06/2025 10:58:24.000 AM (Event 43445 - node.exe launch) and 21/06/2025 10:58:27.000 AM (Event 43448 - script creation, Event 43450 - cmd.exe launch by node.exe)
  • MITRE ATT&CK:
    • Tactic: Initial Access (TA0001)
      • Technique: Supply Chain Compromise (T1195) - Compromise Software Supply Chain (T1195.002).
    • Tactic: Execution (TA0002)
      • Technique: Command and Scripting Interpreter (T1059) - Nodejs (T1059.009), Windows Command Shell (T1059.003).
    • Tactic: Persistence (TA0003) - Note: While this phase triggers the mechanism, the persistence itself is set in Phase 3.
      • Technique: Event Triggered Execution (T1546) - InstallUtil (T1546.009) or similar, depending on how postinstall is classified, but primarily via Command and Scripting Interpreter.
  • Affected User: PAW-TOM\itadmin-tom (The user whose action triggered the installation)
  • Affected Asset: paw-tom (Processes: powershell.exe (root), node.exe, cmd.exe; File System: creation of postinstall.ps1)
  • IOCs (Specific to this Phase):
    • NPM Package: healthchk-lib@1.0.1
    • CommandLine (node.exe): "C:\\Program Files\\nodejs\\node.exe" "C:\\Program Files\\nodejs/node_modules/npm/bin/npm-cli.js" install healthchk-lib@1.0.1

    • TargetFilename (File Created): C:\\Development\\node_modules\\healthchk-lib\\scripts\\postinstall.ps1 (Created by node.exe ProcessGuid {c5d2b969-9050-6856-e101-000000002a01})

Phase 2: Encoded Command Execution & Malware Download Attempt

  • Title: Obfuscated Script Execution and Attempted External Payload Retrieval
  • Adversary Step Description: The cmd.exe process launched in Phase 1 executes the postinstall script by launching a hidden PowerShell process (powershell.exe -NoP -W Hidden -EncodedCommand ...). The encoded command within this PowerShell process attempts to download a malicious executable (SystemHealthUpdater.exe) from the suspicious domain global-update.wlndows.thm using Invoke-WebRequest and save it to the user's AppData directory. A DNS query for the malicious domain is observed during this phase, resolving to ::ffff:127.0.0.1 (localhost), indicating the download attempt likely failed to reach the intended remote server but highlighting the communication attempt. The execution of the encoded command also results in the creation of a temporary PowerShell script file.

Phase 3: Establishing Persistence

  • Title: Registry Run Key Persistence Installation
  • Adversary Step Description: The PowerShell process from Phase 2 proceeds to modify the user's registry hive. It adds a new value named "Windows Update Monitor" to the HKCU\Software\Microsoft\Windows\CurrentVersion\Run key. The data of this registry value is another Base64 encoded PowerShell command (powershell.exe -NoP -W Hidden -EncodedCommand UwB0AGEAcgB0AC0AUAByAG8AYwBlAHMAcwAgACcAQwA6AFwAVQBzAGUyc...). This command, when decoded, is Start-Process 'C:\Users\Administrator\AppData\AppData\Roaming\SystemHealthUpdater.exe'. This ensures that the intended malware file path will be executed hiddenly via PowerShell every time the user logs onto the system.
  • Timestamp (Observed): 21/06/2025 10:58:29.000 AM (Event 43454)
  • MITRE ATT&CK:
    • Tactic: Persistence (TA0003)
      • Technique: Boot or Logon Autostart Execution (T1547) - Registry Run Keys / Startup Folders (T1547.001).
    • Tactic: Execution (TA0002)
      • Technique: Command and Scripting Interpreter (T1059) - PowerShell (T1059.001) (The command stored in the registry).
    • Tactic: Defense Evasion (TA0005)
      • Technique: Obfuscated Files or Information (T1027) - Encoded Content (T1027.003) (The registry value command is encoded).
  • Affected User: PAW-TOM\itadmin-tom (Specifically their registry hive)
  • Affected Asset: paw-tom (Process: powershell.exe; Registry: modified user hive)
  • IOCs (Specific to this Phase):
    • Registry Path: HKCU\Software\Microsoft\Windows\CurrentVersion\Run (General path)
    • Registry Value Name: Windows Update Monitor
    • TargetObject (Sysmon EID 13): HKU\S-1-5-21-...\Software\Microsoft\Windows\CurrentVersion\Run\Windows Update Monitor (Specific path including user SID)
    • Registry Value Data: powershell.exe -NoP -W Hidden -EncodedCommand UwB0AGEAcgB0AC0AUAByAG8AYwBlAHMAcwAgACcAQwA6AFwAVQBzAGUyc... (Full value from Event 43454)
    • Decoded command from registry value: Start-Process 'C:\Users\Administrator\AppData\Roaming\SystemHealthUpdater.exe'
  • Splunk Hunting Steps: (This remains the same as in the previous detailed plan, linking to the PowerShell ProcessGuid identified in Phase 1)
    1. Search for registry modifications to the Run key: Use EventCode=13 targeting the TargetObject pattern for the Run key and value name "Windows Update Monitor". Link to the malicious PowerShell ProcessGuid from Phase 1 Step 4.

The approach for determining scope remains the same: use the key IOCs from all three phases in broadened Splunk queries (removing specific host/user/ProcessGuid filters) to find all instances across your environment.
