Scenario

You work as a Tier 1 Security Analyst L1 for a Managed Security Service Provider (MSSP). Again, you're tasked with monitoring network alerts.

An alert triggered: Misc activity, A Network Trojan Was Detected, and Potential Corporate Privacy Violation.

The case was assigned to you. Inspect the PCAP and retrieve the artifacts to confirm this alert is a true positive.

Your tools:

  • Brim
  • Network Miner
  • Wireshark

  • What was the alert signature for A Network Trojan was Detected?

    I opened the pcap in Brim, From there there are a couple of inbuilt queries

Once you click on it, you can see the Alerts that have been triggered by Suricata Alerts, From there, right click on the “A Network Trojan was Detected” then select search with this as a filter

You can already see the alert signature that triggered this on the far right column, You can also double click on it to open it and see in detail

  • What was the alert signature for Potential Corporate Privacy Violation?

    Follow the same logic as previous one,
  • What was the IP to trigger either alert? Enter your answer in a defanged format.

    Both of these were from the same src IP, as seen from the alert
  • Provide the full URI for the malicious downloaded file. In your answer, defang the URI.

    From BRIM, I can use the following filter to get only time, src ip dest ip host and uri
    _path=="http" | cut ts, id.orig_h, id.resp_h, id.resp_p, method,host, uri | uniq -c | sort ts
    Brim also has inbuilt Filters to see HTTP Requests
    From there, I sorted by time and looked at the first reqeust to download a file
  • What is the name of the payload within the cab file?

    To get this information we need to get the file out, To do this, I moved to Wireshark and located the HTTP stream with the file, since I already know the file name
    Then in File-Export Objects I can choose the HTTP packet with the file and save it, then I can calculate its SHA256 sum and use VT to get more information on that file.

Now we know that its a malicious DLL.

  • What is the user-agent associated with this network traffic?

    Since I am already on the HTTP stream where I foun dhte request for the maliicous file, I can just look at the HTTP Request to see the user agent
  • What other domains do you see in the network traffic that are labelled as malicious by VirusTotal? Enter the domains defanged and in alphabetical order. (format: domain[.]zzz,domain[.]zzz)

    I looked at Statistics - HTTP - Requests to see the number of URLS, then from there, since there weren't many, i checked VT to see which are malicious
  • There are IP addresses flagged as Not Suspicious Traffic. What are the IP addresses? Enter your answer in numerical order and defanged. (format: IPADDR,IPADDR)

    Back to Brim for this, under the built-in Queries - Suricata Alerts by src and dst, we can see some IP addresses categorized as Not suspicious Traffic
