üîê What is JWT?

JWT (JSON Web Token) is a token format often used for authentication. It has three parts:

  1. Header ‚Äî contains metadata, like the algorithm used to sign.
  1. Payload ‚Äî contains user info like username, exp, etc.
  1. Signature ‚Äî proves the token wasn‚Äôt tampered with (if signed correctly).

Format:

base64(header).base64(payload).signature

üß® What is the alg: none vulnerability?

Originally, JWT spec allowed the alg to be "none", meaning the token would skip signature verification. This was useful for debugging but extremely dangerous in production.

If the backend server blindly trusts tokens with "alg": "none" and skips verification:

‚û§ Anyone can forge a token with:

  • "alg": "none"
  • A payload like { "username": "admin" }
  • And no signature

üß™ What you did:

  1. You took a real JWT and decoded it.
  1. You replaced the header to:
    { "typ": "JWT", "alg": "none" }
  1. You modified the payload to:
    { "username": "admin", "exp": 1750715828 }
  1. You left the signature empty.

    Final token looked like:

    <base64_header>.<base64_payload>.
  1. You sent this token to the web application (as a cookie or header).

‚úÖ If the app accepted this token:

  • It did not verify the signature (bad!).
  • It simply trusted whatever data was in the payload.
  • You effectively bypassed authentication and became admin.

üîí How to fix this vulnerability:

  • Always validate JWT signatures properly.
  • Reject any tokens with alg: none.
  • Use strong, secret keys.
  • Use libraries that disallow alg: none by default (most do now).

üîç SSRF (Server-Side Request Forgery)

What it is:

  • An attacker tricks the server into making HTTP requests on their behalf to internal or external systems.

üìå Key Points:

  • Server fetches a URL or resource based on user input.
  • Attacker changes the URL to target internal services (e.g. localhost, 127.0.0.1, AWS metadata).
  • Can lead to:
    • Internal IP/port scanning
    • Access to internal admin panels
    • Data theft (e.g. AWS keys from metadata)

  "url": "http://169.254.169.254/latest/meta-data/"

‚úÖ Prevention:
