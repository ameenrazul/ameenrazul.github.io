Core Windows Processes

Task Manager is a built-in GUI-based Windows utility that allows users to see what is running on the Windows system. It also provides information on resource usage, such as how much each process utilizes CPU and memory. When a program is not responding, the Task Manager is used to terminate the process.

A Task Manager provides some of the Core Windows Processes running in the background. Below is a summary of running processes that are considered normal behaviour.

Note: ">" symbol represents a parent-child relationship. System (Parent) > smss.exe (Child)

  • System
  • System > smss.exe
  • csrss.exe
  • wininit.exe
  • wininit.exe > services.exe
  • wininit.exe > services.exe > svchost.exe
  • lsass.exe
  • winlogon.exe
  • explorer.exe

In addition, the processes with no depiction of a parent-child relationship should not have a Parent Process under normal circumstances, except for the System process, which should only have System Idle Process (0) as its parent process.

SysInternals

  • TCPView - Networking Utility tool.
  • Process Explorer - Process Utility tool.

TCPView is a Windows tool that shows all active TCP and UDP connections on your system. It displays local and remote IP addresses, connection status, and the process using each connection. It works like a clearer, easier-to-use version of the Netstat command, and also includes a command-line version called Tcpvcon.

https://learn.microsoft.com/en-us/sysinternals/downloads/tcpview

Process Explorer is a Windows tool with two main sections. The top part shows all running processes and who owns them. The bottom part changes based on the mode: in handle mode, it shows the files and resources a selected process is using; in DLL mode, it shows the DLLs and files loaded by that process.

Process Explorer enables you to inspect the details of a running process, such as:

  • Associated services
  • Invoked network traffic
  • Handles such as files or directories opened
  • DLLs and memory-mapped files loaded

🖥️ Windows Endpoint Monitoring & Logging Overview

In cybersecurity, just observing real-time activity on a machine isn’t enough. We need to log, collect, and analyze events across all endpoints. This helps with detecting threats, auditing activity, and responding to incidents effectively.

📋 Windows Event Logs

  • Windows stores system events in binary log files (.evtx) at:

    C:\Windows\System32\winevt\Logs

  • These logs can’t be read with a text editor, but can be accessed using:
    • Event Viewer (GUI)
    • wevtutil.exe (Command Line)
    • Get-WinEvent (PowerShell)

🛡️ Sysmon (System Monitor)

  • Part of the Sysinternals Suite, Sysmon provides detailed security event logging.
  • Often integrated with SIEM tools (like Splunk, Wazuh, etc.)
  • It logs things like:
    • Process creation
    • File changes
    • Network connections
  • Uses a config file to control what gets logged.
  • Logs are viewable in Event Viewer under Sysmon.

📊 OSQuery

  • Created by Facebook, works on Windows, Linux, macOS, FreeBSD
  • Allows SQL-style querying of system data (processes, users, network, etc.)
  • Example:
    SELECT pid, name, path FROM processes WHERE name='lsass.exe';
  • Use via osqueryi shell in CMD or PowerShell.
  • For managing multiple systems, use Kolide Fleet, a web-based OSQuery manager.

🧠 Wazuh (EDR Platform)

  • Free and open-source Endpoint Detection & Response (EDR) solution.
  • Uses a manager-agent model:
    • Manager controls and receives logs
    • Agent is installed on monitored machines
  • Features:
    • Vulnerability detection
    • Suspicious activity monitoring
    • Data visualization
    • Baseline behavior tracking

Endpoint Log Analysis

🔍 Event Correlation

Definition:

Event correlation is the process of linking related events from multiple sources—like application, endpoint, and network logs—to uncover suspicious or malicious activity.

Why it matters:

Different logs tell different parts of the story. Correlating them helps reconstruct the full scenario.

🧩 Example:

Let’s say a suspicious network connection occurred:

Log SourceDetails Provided
SysmonProcess name, User account, Machine name, Event ID 3 (Network Connection)
FirewallSource/Destination IP & Port, Protocol, Action taken (allow/block)

By correlating both:

  • You can see who ran what process that connected to a remote IP
  • And whether the firewall allowed or blocked it

Correlation connects:

  • IP addresses (src/dst)
  • Ports (src/dst)
  • Protocol
  • Action taken
  • Process name
  • User
  • Hostname

📊 Baselining

Definition:

Baselining means understanding what "normal" activity looks like on your systems.

Purpose:

Once you know what’s normal, it’s easier to spot anomalies—which are often signs of compromise.

✅ Baseline vs ❌ Unusual Activity

✅ Baseline❌ Unusual
Employees log in from London between 9AM–6PMVPN login from Singapore at 3AM
Each employee uses one workstationOne user tries to log into many machines
Access to approved sites like OneDrive, SharePointUploading 3GB to Google Drive
Only apps like Word, Excel, Teams, Chrome installedfirefox.exe found running on multiple workstations

Even small anomalies can be a red flag—and baselining helps you find the needle in the haystack.

With the basics of endpoint monitoring, event correlation, and baselining covered, you're now ready to:

  • Apply these techniques
  • Investigate suspicious activity
  • Act like a real-world Blue Teamer

We know the baseline and identified the rogue process, now we trace the process, and see what it is doing, if there are network connections, we terminate them and isolate any endpoints these services are running on and send them for forensics.

In the simulated threat investigation activity, we have learned the following:

  • Having a baseline document aids you in differentiating malicious events from benign ones.
  • Event correlation provides a deeper understanding of the concurrent events triggered by the malicious activity.
  • Taking note of each significant artefact is crucial in the investigation.
  • Other potentially affected assets should be inspected and remediated using the collected malicious artefacts.
